Create a CHANGELOG.md following Keep a Changelog format for a web framework.

Include:
- Unreleased section with upcoming changes
- v2.3.0 through v2.0.0 (4 releases) with Added, Changed, Fixed, Deprecated, Removed, Security subsections
- v1.5.0 through v1.0.0 (6 releases)
- Each release should have realistic framework-related changes (routing, middleware, templating, ORM, CLI)
- Dates in YYYY-MM-DD format
